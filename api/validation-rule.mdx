## Overview

The `NotBlacklistedEmail` class is a Laravel validation rule that checks if an email address domain is not from a disposable or temporary email service. It validates against thousands of known disposable email domains.

## Class Reference

### Namespace

```php

```

### Implementation

Implements `Illuminate\Contracts\Validation\ValidationRule`

## Validation Logic

The validation process:

1. Extracts the domain from the email address
2. Checks against the **whitelist** first - if domain is whitelisted, validation passes immediately
3. Merges the built-in blacklist with custom blacklist from configuration
4. In strict mode, also includes greylist domains in the blacklist
5. Compares the email domain against blacklisted domains
6. Fails validation if a match is found

## Usage

### Class Approach

Use the validation rule as a class instance:

```php
use Veeqtoh\ActiveEmail\Rules\NotBlacklistedEmail;

Validator::make($data, [
    'email' => ['required', 'email', new NotBlacklistedEmail()],
]);
```

**Form Request Example:**

```php
use Veeqtoh\ActiveEmail\Rules\NotBlacklistedEmail;

public function rules(): array
{
    return [
        'email' => ['required', 'email', new NotBlacklistedEmail()],
        'name' => ['required', 'string'],
    ];
}
```

### String Validator Approach

Use the registered string validator:

```php
Validator::make($data, [
    'email' => 'required|email|notblacklisted',
]);
```

**Form Request Example:**

```php
public function rules(): array
{
    return [
        'email' => 'required|email|notblacklisted',
    ];
}
```

## Error Messages

### Default Error Message

If validation fails, the default error message is:

```
Sorry, your email provider is not supported. If you think this is an error, please contact us.
```

### Custom Error Message

You can customize the error message in the `config/active-email.php` file:

```php
return [
    'error_message' => 'Please use a valid email address from a trusted provider.',
    // ...
];
```

Or set it via environment variable in your `.env` file:

```env
DISPOSABLE_EMAIL_ERROR_MESSAGE="Please use a valid email address from a trusted provider."
```

## Configuration

The validation rule respects these configuration options:

- `active-email.blacklist` - Custom domains to blacklist
- `active-email.greylist` - Domains to blacklist only in strict mode
- `active-email.whitelist` - Domains to always allow (bypasses blacklist)
- `active-email.strict_mode` - Whether to include greylist in validation
- `active-email.error_message` - Custom validation error message

See the [Configuration](/guides/configuration) guide for details.
